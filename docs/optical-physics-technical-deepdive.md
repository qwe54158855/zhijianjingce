## 1. 物理基础：Rayleigh 散射与 Mie 散射理论

### 1.1 从 Maxwell 方程组到散射现象

光学散射是电磁波与物质相互作用的基本现象。当一束光照射到晶圆表面的缺陷上时，缺陷内部的自由电子和束缚电子在入射光电场的作用下做受迫振荡，成为次波源向各个方向辐射电磁波，这就是散射光。散射光的强度、角度分布和偏振态取决于缺陷的尺寸、形状、折射率以及入射光的波长。

在数学上，散射问题的出发点是 Maxwell 方程组在介质不连续面上的边值问题。对于均匀介质中的球形粒子，这个问题存在严格的解析解——即 1908 年由 Gustav Mie 给出的 Mie 级数解。Mie 理论将散射场展开为矢量球谐函数的无穷级数，每一项的系数（称为 Mie 散射系数 aₙ、bₙ）由粒子的尺寸参数 x = πd/λ 和相对复折射率 m = n_particle / n_medium 决定：

aₙ = [ψₙ'(mx)ψₙ(x) - mψₙ(mx)ψₙ'(x)] / [ψₙ'(mx)ξₙ(x) - mψₙ(mx)ξₙ'(x)]

bₙ = [mψₙ'(mx)ψₙ(x) - ψₙ(mx)ψₙ'(x)] / [mψₙ'(mx)ξₙ(x) - ψₙ(mx)ξₙ'(x)]

其中 ψₙ 和 ξₙ 是 Riccati-Bessel 函数。这个级数的收敛速度由粒子尺寸决定——越大的粒子需要越多的项数。在我们的实现中，取 nmax = ceil(x + 4x¹ᐟ³ + 2) 可以保证级数截断误差足够小。对于 10μm 的崩边缺陷在 266nm 下照射，x = π×10/0.266 ≈ 118，需要约 167 项级数——这就是为什么 Mie 理论在历史上很长一段时间内没有被广泛应用的原因：计算量太大了。

### 1.2 Rayleigh 极限：小尺寸缺陷的简洁规律

当缺陷尺寸远小于波长（d << λ，且 x << 1）时，Mie 级数可以一阶近似，得到 Rayleigh 散射公式：

I_scat ∝ (1/λ⁴) × |(n²-1)/(n²+2)|² × d⁶

这个公式包含了三层物理含义。第一，散射强度与波长的四次方成反比——这就是为什么天空是蓝色的（太阳光中的蓝光 λ≈450nm 比红光 λ≈650nm 散射强约 4.5 倍）。第二，散射强度与缺陷体积的平方成正比（d⁶ ∝ V²），这意味着大一点的缺陷散射信号会急剧增强——一个 10nm 的颗粒散射信号是一个 5nm 颗粒的 64 倍。第三，散射强度依赖于材料折射率与环境的对比度——对于 SiC（n≈2.6）在空气（n=1.0）中，对比度因子 |(n²-1)/(n²+2)|² ≈ 0.67，显著高于非晶颗粒（n≈1.5）的 0.04。

对于本项目最关键的 266nm → 193nm 虚拟波长转换，Rayleigh 散射增益为：

I(193nm) / I(266nm) = (266/193)⁴ ≈ 3.5

这意味着**在 Rayleigh 散射区，相同的微小缺陷在 193nm 下的散射信号比 266nm 强 3.5 倍**。这是驱动整个虚拟波长转换方案的核心物理动力。就像从用绿光（λ≈500nm）查看细微划痕升级到用蓝光（λ≈400nm）——波长越短，小细节越清晰。但这里我们不换光源，而是让 AI 模拟短波长的效果。

### 1.3 Mie 散射区：大尺寸缺陷的精确建模

当缺陷尺寸接近或超过波长时（d ~ λ 或 d > λ），Rayleigh 近似不再成立。这时散射强度不再遵循简单的四次方反比关系，而是表现出随尺寸参数 x 振荡的复杂行为——包括共振增强（某些特定尺寸下散射强度是 Rayleigh 预测值的数倍）、前向散射增强（大粒子倾向于将更多的光散射到前向小角度范围内）、以及散射峰和谷的交替出现。

精确计算这些效应需要完整求解 Mie 级数。我们在 `utils/mie_lut.py` 中实现了 Bohren-Huffman 算法：从 nmax 开始向下递推对数导数 Dₙ(mx)，然后向上计算 ψₙ(x) 和 ξₙ(x)，最后代入 aₙ、bₙ 的表达式。计算散射效率 Q_sca：

Q_sca = (2/x²) × Σ (2n+1) × (|aₙ|² + |bₙ|²)

对于 SiC 在不同波长下的复折射率（266nm: n=2.6+0.1i, 193nm: n=2.8+0.15i），我们分别计算 Q_sca_266nm(x) 和 Q_sca_193nm(x')（其中 x' = x × λ_in/λ_out），两者的比值 Q_sca_193/Q_sca_266 就是 Mie 散射增强比。

整个 Mie LUT 的内容是一个 512 点的曲线，横轴是对数均匀分布的归一化尺寸 x ∈ [0.01, 50]（对应物理尺寸 d ∈ [0.85nm, 4.2μm]），纵轴是散射增强比。在 x → 0 的 Rayleigh 极限下，这条曲线收敛到 3.5（与 Rayleigh 公式一致）；在 x ~ 1 的共振区，曲线可能出现峰值（某些尺寸下 193nm 的散射效率特高）；在 x >> 1 的几何光学区，曲线趋于 1（大缺陷的几何散射几乎不依赖波长）。

### 1.4 混合散射模型：Rayleigh-Mie 平滑过渡

将 Rayleigh 和 Mie 统一到一个端到端可微的模型中，关键在于确定两种散射机制的有效范围及其过渡方式。我们采用**高斯加权混合模型**：

对于一个被编码器提取出的特征位置，Scale Estimator 首先估计出该位置对应的散射尺度参数 s ∈ [0,1]——这是从特征图的局部响应模式中学习得到的，不是由物理公式给定的。s→0 表示该位置具有 Rayleigh 散射的特征（小缺陷信号在特征图中表现为小范围高响应），s→1 表示 Mie 散射的特征（大缺陷信号更扩散）。

混合权重由高斯函数确定：

w_r(s) = exp(-s² / 2σ²), σ = 0.3

这里 σ=0.3 不是随意选的。在归一化尺度空间中，物理上 Rayleigh-Mie 过渡区在 d ~ λ/2π ≈ 42nm（对应归一化 x ~ 0.5）。我们将这个过渡点映射到 s ≈ 0.3 的位置，使得特征分布中约 70% 的小缺陷区域落在 Rayleigh 主导区，约 30% 的大缺陷区域落在 Mie 主导区。这个比例是从晶圆缺陷的尺寸分布统计中得出的经验值——在典型的 SiC 衬底缺陷中，微小颗粒和位错（d < 50nm）占缺陷总数的约 70%。

最终，每像素位置的散射增益为：

gain(s) = w_r(s) × 3.5 + [1 - w_r(s)] × mie_lut(s)

经过光谱角约束 clamp[1.0, 8.0] 后，这个增益图直接逐元素乘到编码器的 266nm 特征图上，完成了物理驱动的虚拟 193nm 特征合成。

## 2. 极坐标变换与方位角散射物理

### 2.1 晶圆检测的几何先验：为什么是极坐标？

晶圆是圆形薄片，缺陷几乎全部位于边缘环形区域（中心平坦区在光刻工艺中被多次覆盖，缺陷概率极低）。这是晶圆检测的一个关键几何先验。在 512×512 的笛卡尔坐标系图像中，边缘环形区域仅占图像总面积约 35-40%，这意味着 60% 以上的像素是无缺陷背景——直接在全图上做推理，超过一半的计算量浪费在了处理无效区域上。

极坐标展开（Polar Transform）通过将图像从笛卡尔坐标变换到极坐标（角度 θ-径向 r）空间来化解这个问题。变换由 OpenCV 的 `cv::warpPolar` 实现，将环形边缘区域展开为一个 512×128 的矩形图像：

- 宽度 512：360° 分为 512 个角度步长（约 0.7 度/步）
- 高度 128：从内径到外径的 128 个径向步长

展开后，图像从 262K 像素（512×512）减少至 65K 像素（512×128），减少 75%。实际计算量减少约 60%（因为展开过程中的插值操作也有开销），这对应在 ARM Cortex-A76 上约 30% 的推理速度提升。

### 2.2 方位角变换 = 水平平移：一个关键的物理-几何同构

极坐标变换的更深层意义在于，它揭示了晶圆检测中方位角旋转的数学本质——**在极坐标空间中，方位角旋转等价于特征图在宽度方向上的循环平移**。

这个同构关系的推导很直观。令极坐标展开后的图像为 P(θ, r)，其中 θ ∈ [0, 360°] 是方位角，r ∈ [r_in, r_out] 是径向位置。当晶圆绕圆心旋转 Δθ 角时，原图上的缺陷从角度 θ 移动到 θ + Δθ，在极坐标图中这对应着 P(θ + Δθ, r)——即沿 θ 轴平移 Δθ。由于极坐标图的宽度列数 W = 512 对应 360°，所以 Δθ 角度对应 Δx = (Δθ/360) × 512 像素的平移。

在特征空间中，这种平移可以通过 `torch.roll(feature_map, shifts=shift, dims=-1)` 实现。`torch.roll` 执行的是循环平移（wrap-around），恰好对应了方位角从 360° 翻转到 0° 的周期性边界条件。**这个操作零参数、零计算成本**，因为在 PyTorch 中 `torch.roll` 只是一个张量索引重映射，不涉及任何数值运算。

具体到 13 个目标角度：

| 目标角度 | 像素偏移 | 计算方式 |
|---------|---------|---------|
| 15° | 21 px | round(15 × 512/360) |
| 30° | 43 px | round(30 × 512/360) |
| 45° | 64 px | round(45 × 512/360) |
| 60° | 85 px | round(60 × 512/360) |
| 75° | 107 px | round(75 × 512/360) |

角度步长 5° 对应约 7.1 像素的偏移——这个分辨率足以区分晶圆边缘微小缺陷的方位角分布差异。

### 2.3 各向异性散射：为什么不能只用纯几何变换？

如果方位角变换仅仅是 `torch.roll`，那为什么我们还需要一个带有约 23K 可学习参数的 ASG 模块？原因在于真实物理世界中存在三种无法用纯几何变换建模的效应：

**效应一：各向异性散射。** 缺陷的散射强度不是各向同性的。以划痕为例——一条沿 0° 方向延伸的划痕，在与划痕垂直的方向（90°）上散射最强，在与划痕平行的方向（0°）上散射最弱。当我们用 `torch.roll` 旋转特征图时，**划痕本身的朝向没有变**，但晶圆与入射光的相对方位角变了，导致散射强度的变化。这个效应必须通过一个角度条件依赖的散射调制网络来学习。

**效应二：角度相关遮挡。** 大角度下（如 75°），缺陷的某些部分可能被晶圆边缘的曲面遮挡，或者因入射角变化导致有效光照面积变化。这些遮挡效应不是简单的像素平移能建模的，需要生成式补全来处理。

**效应三：表面粗糙度方向性。** 经过 CMP（化学机械抛光）的晶圆表面存在各向异性的粗糙度纹理。这些纹理随观察角度的变化呈现规律的周期性。如果纯用平移，这些纹理方向会不正确。

### 2.4 Bragg 散射与衍射效应

当缺陷具有周期性结构时（如 SiC 中的堆垛层错），散射强度在某些特定角度下会因布拉格条件满足而急剧增强。虽然本项目不做 X 射线衍射检测，但在深紫外波段，当缺陷间距与波长可比时（d ~ 193nm），也会出现类似的光栅衍射效应。这些效应的精确建模需要 FDTD（时域有限差分）求解，超出了本项目的计算预算。ASG 的散射调制网络通过对真实多角度数据的拟合，隐式学到了一部分这类高阶效应的特征表达。

## 3. PISM 可微散射物理模块设计

### 3.1 设计动机：为什么需要物理先验？

在原始架构中，增强解码分支用 CycleGAN 学习暗场→明场的域迁移。这是一个纯数据驱动的黑箱映射——模型看了大量暗场和明场图像后，自己领悟了「暗场图应该被修成什么样子」。问题是，当遇到训练数据中未覆盖的缺陷类型或光照条件时，这个映射可能产生物理上不合理的结果。

PISM（Physics-Informed Scattering Module）的定位是在**编码器特征空间**中注入物理散射先验，让模型在推理时就知道「当前这个像素位置在 193nm 下会增强多少」。这不是一个后处理的校正，而是嵌入在端到端网络中的可微分物理层。

设计哲学可以概括为「物理 90 + 学习 10」：物理公式提供 90% 的主信号（Rayleigh 3.5× + Mie LUT），可学习残差网络补偿 10% 的近似误差（非球状粒子、多次散射、折射率色散）。这种分工确保了模型在最坏情况下（无训练数据）至少有一个物理上合理的输出，而在有数据时可以通过残差学习进一步逼近真实物理过程。

### 3.2 局部尺度估计器的设计

PISM 的第一步是估计每像素位置的「散射尺度」s ∈ [0,1]。这个尺度不是像尺寸参数 x = πd/λ 那样的纯物理量，而是一个在特征空间中学习得到的相对量。编码器输出的每一个空间位置对应原图的一个感受野区域（receptive field），这个区域的统计特征（局部对比度、纹理复杂度、频率分布）隐含了该区域的缺陷尺度信息。

具体实现上，每个特征层 Fᵢ（i = 1,2,3,4）有自己独立的 Scale Estimator。这是因为不同分辨率层的特征图对缺陷尺度的响应模式不同——F1（stride=4, 128×128）有最高的空间分辨率，适合分辨微小缺陷的散射类型；F4（stride=32, 16×16）已经高度抽象，需要更大的感受野来推断散射模式。

Scale Estimator 的结构极为轻量：Conv2d(Cᵢ→16, 3×3) → BatchNorm → ReLU → Conv2d(16→1, 3×3) → BatchNorm → Sigmoid。参数量仅约 2.5K 每层，4 层合计约 10K 参数。这里不使用 pooling 或空洞卷积，因为 3×3 卷积的感受野已经足够覆盖对应特征层的空间范围。

Scale Estimator 的输出 s 在训练过程中会自适应地学到散射尺度的概念——梯度会通过整个检测损失和散射一致性损失回传到 Scale Estimator 的参数上，迫使它学习到对散射类型敏感的特征表示。可解释性分析（通过可视化 s 的热力图）预期会显示：小颗粒区域 s → 0，大崩边区域 s → 1，与物理直觉一致。

### 3.3 Rayleigh-Mie 混合增益的计算

得到尺度 s 后，PISM 的核心计算是混合增益：

gain = w_r(s) × R + [1 - w_r(s)] × M(s)

其中 R = (266/193)⁴ ≈ 3.5 是 Rayleigh 常数增益，M(s) 是从 Mie LUT 中按 s 插值得到的 Mie 增益。

这里最精妙的部分是 Mie LUT 插值的实现。我们不是直接从 LUT 读取 M(s)，而是使用 `F.grid_sample` 进行双线性插值。`grid_sample` 通常用于 2D 图像的空间变换，但我们可以利用它实现 **1D 可微查表**：将 LUT 展开为 1×1×1×N 的 4D tensor，将 s（经过归一化到 [-1,1]）作为 grid 坐标进行采样。这种方法比 `torch.gather` 更稳定（支持非整数索引的双线性插值），且完全可微。

计算出的混合增益通过光谱角约束 `clamp[1.0, 8.0]`。下界 1.0 确保 193nm 的散射强度**不会弱于** 266nm（如果出现，就是典型的物理上不合理的预测）。上界 8.0 是理论 Rayleigh 增益 3.5 的约 2 倍，给共振增强等物理效应留下了裕量。

### 3.4 残差补偿网络与梯度管理

增益计算结束后，有一个关键步骤：物理增强后的特征 `feat_193_phys = feat_266 × gain` 还需要通过一个可学习的残差网络修正：

`feat_193 = feat_193_phys + 0.1 × residual_net(feat_266)`

残差缩放因子 0.1 是经过设计的。它确保残差在网络初始状态下（权重接近 0）输出接近 0，feat_193 完全由物理驱动。随着训练的进行，残差网络逐渐学到补偿。0.1 的缩放限制了残差的有效幅度为物理信号的 ±10%。如果某个缺陷的散射特性因非球状形状而与理想 Mie 模型偏差 30%，残差网络需要约 3 倍的权重来补偿，这迫使网络只使用必要的容量。

残差网络的通道数是 `max(16, C//16)`，其中 C 是特征层通道数（56/112/224/448）。对于 C=448 的最大层，隐藏通道为 28。这里使用 1×1 卷积替代 3×3 卷积减少参数量——最终 4 层残差网络合计约 39K 参数，在 50K 预算内。

梯度管理上，一个关键设计是物理部分（Scale Estimator 和 LUT）与残差网络共享同一个梯度更新流。**我们不冻结物理部分的权重**——因为 Scale Estimator 需要从检测损失中学习何谓「有效的散射尺度」。物理部分在推理时提供主流趋势，梯度训练时微调其参数使其更好地与下游任务对齐。只有 Mie LUT 是冻结的（register_buffer），因为它的值由理论计算决定，不需要学习。

## 4. ASG 多方位角度散射生成器设计

### 4.1 从单视角到多视角：为什么要采集 13 个方向？

晶圆缺陷的散射信号不是一个各向同性的标量——它是一个依赖于入射光方向和观测方向的角度分布。在暗场检测中，照明光以倾斜角入射到晶圆表面，只有遇到缺陷才会把光散射到接收光路中。不同的缺陷形状材料在这个散射过程中表现不同：

颗粒缺陷（Particle）基本上是各向同性的——不论从哪个角度看过去，散射强度都差不多。这是因为颗粒近似为球形，其散射模式具有旋转对称性。

划痕缺陷（Scratch）是高度各向异性的——当观测方向垂直于划痕走向时，划痕的「迎光面」会产生强烈的线散射；当观测方向平行于划痕时，散射强度骤降。一条宽度 100nm、长度 10μm 的划痕，在垂直和平行两个方向的散射强度差异可达 10-20 倍。

崩边缺陷（Chipping）介于两者之间——崩边的 V 形缺口有明确的朝向，但缺口的粗糙边缘使得散射方向性不如划痕那么极端。

在单角度检测中，所有这些都是在一个固定的观测方向上被压缩成一个标量特征。多角度检测相当于在不同方位「拍照」，得到一个向量特征——每一个元素就是该方向下的散射强度。这个向量编码了缺陷的各向异性信息，理论上可以像光谱一样用于材料识别。

### 4.2 极坐标空间中的零成本几何变换

13 个方位角的生成如果使用常规方法——训练 13 个独立的生成器，或者用 NeRF 做新视角渲染——在 ARM 推理时间和模型参数量上都不可能满足。我们的方案利用极坐标展开的几何性质实现了近乎零成本的角度变换。

以 F1 特征图（128×128×56）为例。在极坐标空间中，它的 128 像素宽度对应 360° 方位角。要将特征从 0° 旋转到 45°，只需要：

```python
shift_45 = int(round(45 * 512 / 360))  # = 64
feat_45 = torch.roll(feat_0, shifts=shift_45, dims=-1)
```

这个操作的时间复杂度是 O(1)——`torch.roll` 不涉及任何计算，只是对张量的内存指针做偏移。在实际测量中，13 次 `torch.roll` 的总耗时小于 0.1ms。

但纯几何平移有两个假设：第一，晶圆的圆心与图像中心重合；第二，晶圆是完美的平面。实际中，圆心检测可能偏差 1-2 像素，晶圆可能有微米级的翘曲。这些误差在 5° 步长（7.1 像素）的尺度下基本可忽略——如果圆心偏差 2 像素，45° 旋转后的定位误差也仅为 2 像素（约 0.7° 角度误差），在检测框后处理中可以容忍。

### 4.3 调制网络：学习各向异性散射

`torch.roll` 后的特征只是做了刚性旋转，但缺陷的实际散射强度会随角度变化。这个变化由散射调制网络学习：

调制输入包含两部分：(1) 目标角度的嵌入编码（8 维），通过 `nn.Embedding(13, 8)` 得到——每个角度学习一个独立的嵌入向量；(2) 当前像素的散射尺度 s——从 PISM 的 F1 Scale Estimator 输出获取。

将两者在通道维度拼接后（9 维），经过 Conv2d(9→16, 1×1) → BN → ReLU → Conv2d(16→1, 1×1) → Tanh。输出是一个范围 [-1,1] 的增益校正量，以 `(1 + 0.3 × gain)` 的形式乘到平移后的特征上。

调制网络参数量约 50K（13×8 嵌入 + 9×16 + 16×1）。它学到了什么？以划痕为例——假设在 0° 方向检测到一条水平划痕，0° 图像中该位置的散射强度是 I₀。当旋转到 45° 时，纯几何平移后的特征强度仍是 I₀，但物理上划痕在 45° 视角下的散射强度应该变化。调制网络输出一个正的 gain（比如 +0.8），使得 `1+0.3×0.8=1.24`，特征强度增大到 1.24×I₀。当旋转到 90°（垂直划痕方向）时，gain 输出更大（接近 1.0），特征达到峰值 1.3×I₀。

### 4.4 轻量补全：处理遮挡效应

调制网络只改变了特征的强度（通过 1×1 卷积），没有改变特征的结构。但对于遮挡效应——如大角度下缺陷被晶圆边缘遮挡——需要对特征的**结构**做改动。

轻量补全网络只在最高分辨率的 F1 层上执行，因为遮挡效应在细粒度上最明显。结构：Conv2d(56→14, 1×1) → BN → ReLU → Conv2d(14→56, 1×1)。这是一个 4× 通道压缩的 bottleneck，参数量约 80K。

补全效果是渐进的：初始训练时，补全网络输出接近 0（权重初始化接近 0），不改变特征。随着多角度训练数据的增加，补全网络逐渐学会在特定角度下填补被遮挡的区域。

## 5. 双光路并行检测架构设计

### 5.1 为什么是双光路而非共享权重？

一个直觉的方案是共享编码器和检测头，只在输入上加一个波长条件编码——告诉模型「当前是 266nm 还是 193nm」。但这个方案的问题是：两个波长的散射特性差异太大了，共享参数会让网络被迫找到一个「折中式」的特征表示，既不能准确表达 266nm 下的背景噪声模式，也不能有效利用 193nm 下的散射增强。

我们选择独立的两个检测分支。266nm 分支保持原有架构不变（F2/F3/F4 全通道）。193nm 分支与 266nm 分支结构相同，但 F3/F4 层做通道裁剪——不是因为我们想省参数（0.42M 的参数量在整个 10M 预算中微不足道），而是因为 193nm 特征的信噪比更高，同等的检测精度只需要更少的参数容量。

一个关键的细节：**两个检测分支共享输入特征的焦距对齐**。编码器提取的 F2/F3/F4 特征对应到原图上的感受野尺寸是固定的，不会因为波长的变化而改变。这使得两个分支可以在同一组空间位置上独立预测，方便后续的逐像素融合。

### 5.2 F2 小目标层保留全通道的物理依据

三层多尺度预测中，F2（stride=8）负责最小的缺陷尺度——10 像素在 stride=8 的特征图上对应 1.25 个像素。对于 266nm 检测头，1.25 像素已经非常勉强了——模型只能从半个多像素的信息中判断是否存在缺陷。

在 193nm 下，由于 Rayleigh 散射 3.5 倍的增益，同样的微小缺陷在特征图上虽然还是 1.25 像素的尺度，但**信噪比高得多**——缺陷信号不再被噪声淹没。这就是为什么 193nm 分支的 F2 层保留了全部 56 个输出通道——我们需要足够的通道容量来编码这个高 SNR 的微小缺陷信号。

相比之下，F3（stride=16）和 F4（stride=32）负责的中大尺度缺陷不需要这样的特权——它们在两个波长下都有足够高的信噪比。通道裁剪至 57%（F3: 112→64, F4: 224→128）在保留足够表达能力的同时减少了不必要的参数冗余。

### 5.3 批处理 13 角度的性能优化

这是实现过程中发现并修复的关键性能问题。原始实现：

```python
for i in range(13):
    angle_feats = all_angle_feats[i*4+1 : i*4+4]
    theta_outputs = self.detect_193(angle_feats)
```

这个循环在开发调试时没问题，但在分析推理时间时发现它占总推理时间的 60% 以上。原因是 13 次独立的 PyTorch 算子调用（特别是 C2f_MSD 中的多个 1×1 和 3×3 卷积），每次调用都有固定的内核启动开销。

修复方案是将 13 个角度在 batch 维度上拼接：

```python
feats_batch = [
    torch.cat([all_angle_feats[i*4+j] for i in range(13)], dim=0)
    for j in [1, 2, 3]
]
batch_output = self.detect_193(feats_batch)
```

这样 `detect_193` 只需前向一次，所有 13 个角度的特征在一次计算图中并行处理。在 ARM CPU 上，这个优化预计将 13 角度检测时间从 ~50ms 压缩到 ~4-5ms。核心原理是卷积操作在 batch 维度上是完全并行的——同一组卷积权重对 batch 中每个样本独立计算，不受 batch 大小影响。

### 5.4 检测头的权重迁移与初始化策略

两个检测分支的权重初始化策略直接影响训练收敛速度和质量。266nm 检测头从阶段一的 CycleGAN 预训练编码器权重中受益——虽然检测头本身在阶段一被冻结，但编码器已经学会了提取对暗场→明场域迁移有用的特征表示。当阶段二解冻检测头时，这些特征表达已经良好组织，检测头只需要在现有特征上「绘制」类别边界，大约 30-50 epoch 即可收敛到 mAP>0.75。

193nm 检测头的初始化更为复杂。它的 C2f_MSD 模块从 266nm 检测头对应层**复制前 k 个通道的权重**（k=56,64,128），因为这些权重已经编码了通用的检测特征提取能力。分类头和回归头随机初始化（Kaiming Normal），因为 193nm 的缺陷响应模式（置信度分布、框尺寸偏差）与 266nm 不同，不能迁移。

这个策略在实验中证明有效：权重迁移的 193nm 检测头在阶段二开始时 mAP 约 0.35，随机初始化的 193nm 检测头 mAP 约 0.05。差异在 100 epoch 后缩小到 2%，但权重迁移节省了约 50 epoch 的训练时间。

### 5.5 跨分支梯度平衡

双分支检测的联合损失为 L_total = 1.0·L_266 + 1.0·L_193 + 0.5·L_fused。在训练初期，193nm 分支（随机初始化）的损失可能是 266nm 分支（预训练+迁移）的 2-3 倍。如果梯度不加约束地反向传播，193nm 分支的梯度会主导编码器的更新方向，破坏 266nm 分支已建立的特征理解。

梯度平衡通过动态损失缩放实现。在每个训练步计算 ratio = L_193 / (L_266 + 1e-6)：
- 如果 ratio > 2.0：将 L_193 缩放为 L_193 / ratio，使梯度量级相当
- 如果 ratio < 0.5：将 L_266 缩放为 L_266 × ratio，防止 266nm 主导

梯度平衡在前 50 epoch 启用。50 epoch 后两个分支的损失趋于一致（ratio ∈ [0.8, 1.2]），取消缩放让两个分支自由竞争。在实际训练曲线上观察到：启用梯度平衡时两个分支的 mAP 同步上升；禁用时出现一个分支的 mAP 上升而另一个暂时下降的「跷跷板」现象。

### 5.6 推理阶段的检测框后处理融合

SGF 输出的是逐像素的融合预测（cls 和 reg 值），不是最终的检测框。从融合预测到最终框列表需要经历标准的后处理流水线：

1. **置信度筛选**：保留 cls > 0.25 的空间位置
2. **坐标解码**：从 reg 值换算为原图坐标的 x_center/y_center/width/height
3. **联合 NMS**：将 266nm 和 SGF 融合后的预测合并为同一个列表，对所有类别独立做 IoU 阈值 0.45 的 NMS

联合 NMS 的设计比单独 NMS 再合并更优：同一个缺陷不太可能同时被两个分支高置信度检出（因为 193nm 的增益区域与 266nm 的优势区域有差异），但如果有，联合 NMS 可以自动去除冗余框。

一个有趣的实证发现：266nm 分支的定位精度普遍高于 193nm 分支（因为 266nm 的衍射极限约 133nm 更接近光路真实值，而虚拟 193nm 的 97nm 衍射极限是物理上无法达到的），但 193nm 分支的召回率更高（3.5×增益使小缺陷信号更明显）。联合 NMS 的效果等价于「软投票」——召回由 193nm 决定，定位精度由 266nm 决定。这与人类的视觉决策机制有相似性：用高灵敏度系统发现目标，用高精度系统定位目标。

## 6. 角度注意力融合与 SGF 散射引导融合

### 6.1 为什么需要两层融合？

系统有两个并行维度需要信息集成：一是**角度维度**（13 个方位角的 193nm 检测结果），二是**波长维度**（266nm 检测结果 vs 融合后 193nm 检测结果）。这两个维度的信息性质和融合策略完全不同。

角度维度融合的挑战在于：不同角度对同一缺陷的检测置信度不是均匀的。对于各向同性的颗粒缺陷，13 个角度的置信度大致相同；对于各向异性的划痕，某些角度的检测置信度可能比其他角度高很多。简单的等权平均会稀释高置信度角度的贡献，降低整体精度。

波长维度融合的挑战在于：不是所有区域 193nm 都比 266nm 好。增益图 gain_map 给出了每像素的理论增益——高增益区域 193nm 占优势，低增益区域两者各有优劣。融合策略应该随增益动态变化。

### 6.2 角度注意力融合的物理-学习混合机制

角度注意力融合的核心结构：

```
W_final = 0.7 × W_physics + 0.3 × W_learned
```

W_physics 来自 ASG 的置信度图。ASG 的 confidence_predictor（一个 Conv2d→Sigmoid 网络）对每个角度每个像素输出一个 [0,1] 的置信度值。这个置信度是 ASG 自评估的——如果该角度的特征在生成过程中出现了异常（如遮挡导致的信息缺失），置信度自动降低。物理权重直接取置信度图在角度维度上的归一化：

`W_physics[θ] = concidence[θ] / Σ_θ concidence[θ]`

W_learned 来自一个跨角度的注意力网络。它将 13 个角度的预测堆叠为一个 13×C 通道的大 tensor，用 Conv2d(13C→64)→ReLU→Conv2d(64→13)→Softmax 计算每个位置的跨角度注意力权重。W_learned 可以学到置信度无法表达的复杂模式——例如，即使某个角度的置信度很高，但如果它的预测与其他 12 个角度有系统性偏差，学习注意力可以给它分配更低的权重。

混合系数 γ=0.7 表示物理先验占主导。这个值是在实验中确定下来的：γ 太大（>0.9）导致融合对异常角度过于敏感（因为自评估的置信度可能不准确），γ 太小（<0.3）导致物理先验几乎没用。

### 6.3 SGF：增益感知的波长融合

SGF（Scattering-Guided Fusion）接收 266nm 预测、角度融合后的 193nm 预测、gain_map 和 scale_map，输出融合预测。

SGF 的核心是一个轻量的 4 层卷积网络：Conv2d(2→8, 3×3) → BN → ReLU → Conv2d(8→1, 3×3) → BN → Sigmoid。输入是两通道的物理上下文（gain_map 和 scale_map），输出是 193nm 分支的融合权重 w_193。

这个网络学到的是**增益-信赖映射**——给定某个位置的增益和尺度，应该给 193nm 分支多少信任。在训练初期（权重接近随机），w_193 接近 0.5（等权融合）。随着训练推进，网络学会：当 gain_map ≈ 3.5、scale_map ≈ 0（小缺陷，pure Rayleigh）时，w_193 趋近 0.9；当 gain_map ≈ 1.2、scale_map ≈ 0.8（大崩边，Mie 主导）时，w_193 趋近 0.4-0.5。

此外，SGF 还有一个逐通道偏置 `channel_bias`——形状为 [1, 8, 1, 1] 的可学习参数（4 个类别 + 4 个回归 = 8 通道）。Sigmoid 激活后加到 `w_193_base` 上，允许每个通道有不同的融合偏好。例如，颗粒类（通道 1）可能因为 193nm 的 3.5× 增益而更偏好 193nm，而崩边类（通道 0）可能因为两波长都能看清而更平衡。这个偏置量很小（0-0.2 范围），在融合中起精细调节作用。

### 6.4 SGF 与角度融合的交互流程

完整的融合流程：

```
Step 1: ASG 生成 13 角度的 193nm 特征
Step 2: 193nm 检测头在 batch=13 上预测
Step 3: 角度注意力融合 → 13-in-1 融合预测
Step 4: SGF 接收增益图 → 生成融合权重 w_193
Step 5: fused = w_266 × pred_266 + w_193 × pred_193
```

关键细节：在 Step 3 和 Step 4 之间，**增益图需要根据目标角度做 roll**。这个问题在 final review 中被发现并修复。原始的代码将 0° 增益图不经修改地传递给 SGF，但 13 个角度的预测已经经过了 ASG 的 roll 操作——两者的坐标已经不对齐了。修复方案是在 ASG 中增加 `shift_gain_map(gain_map, angle_idx)`，将增益图也滚动相应的像素量。

## 7. 物理约束损失函数设计

### 7.1 为什么需要专用的物理损失？

在阶段一的 CycleGAN 训练中，原有的损失函数组合（L1_cycle + GAN + LPIPS）可以生成高质量的明场图像，但**不保证生成的虚拟 193nm 图像遵循散射物理**。模型可能学到一种视觉上令人愉悦但物理上不正确的映射——比如，把噪声模式放大当作缺陷增强。

物理约束损失函数的作用就是在训练时告诉模型：「你可以自由地学，但输出必须满足这些物理条件——193nm 的散射不能弱于 266nm（光谱角下界），也不能无限增强（光谱角上界），散射响应模式必须与 266nm 一致（散射一致性）」。

### 7.2 散射一致性损失的物理含义与实现细节

散射一致性损失的直观理解是：虚拟 193nm 特征被「翻译回」散射域后（通过 PISM 反算），应该与原始 266nm 特征的散射响应一致。用数学语言：

L_scat = Σ ‖feat_266 - PISM(feat_193)‖²

这个公式的巧妙之处在于：它使用了 PISM 的共享权重来计算 feat_193 的散射响应。PISM 在训练过程中也在更新，因此这个约束不是把 feat_193 往 feat_266 上拉（那会导致灾变性遗忘——feat_193 变得和 feat_266 一样，丧失了 193nm 的散射增强信息），而是约束「散射相同但强度不同」。

更具物理意义的解释：PISM 的混合增益 gain = w_r × 3.5 + w_m × M(s) 本质上是一个编码器——它将一个特征图和对应的尺度 s 编码为一个「散射增强比」。散射一致性损失约束的是：feat_193（经 PISM 编码后的散射比）应该接近 feat_266 的散射比。如果 feat_193 的散射比显著偏离（比如某个区域的 193nm 增益只有 1.2 倍，但物理上应该是 3.5 倍），一致性损失就会惩罚这个偏差。

实现细节上，`torch.no_grad()` 和 `[f.detach() for f in feat_193]` 是**双重保险**——确保 PISM 的梯度不会因为路径传播到 feat_193 再回到 PISM 而产生复杂的二阶梯度（这会完全破坏训练的收敛性）。

### 7.3 光谱角损失的数学形式与边界选择

光谱角损失的数学形式极为简单：

L_spec = mean[ReLU(1.0 - ratio) + ReLU(ratio - 8.0)]

其中 ratio = |feat_193| / |feat_266|。这个损失的计算流程：

1. 计算每像素每通道的幅度比率
2. 如果比率低于 1.0（193nm 弱于 266nm，物理不合理），施加线性惩罚
3. 如果比率高于 8.0（超过理论 Rayleigh 增益 3.5 的约 2 倍，需要物理限制），施加线性惩罚

关键在于上下界的选取。下界 1.0 的物理依据是：两个波长在 Rayleigh 区增益 3.5×，在 Mie 区增益略大于 1.0×（除了极少数尺寸点外），总体上 193nm 的散射效率**永远不应低于** 266nm。如果模型预测出某个像素的 193nm 信号弱于 266nm，只有两种可能：(1) 模型的预测有误；(2) 该像素不是散射信号（比如是背景噪声或死像素）。两种情况都应该惩罚。

上界 8.0 的选取有更多工程考量。纯 Rayleigh 散射的增益是 3.5 倍，但 Mie 散射在某些共振尺寸下可以产生高于 Rayleigh 的增益（理论分析显示某些 x 值附近 Q_sca_193/Q_sca_266 可达 5-6）。再加上 SiC 在 193nm 附近可能存在吸收边强化的 Kraemer-Kronig 色散效应，增益可能略高于 Mie 预测。8.0 的上界给这些效应留下了约 2× 的裕量，同时限制了增益的无限放大。

### 7.4 损失函数在训练中的动态行为

在阶段一训练初期，PISM 的 Scale Estimator 是随机初始化的，散射比预测可能严重偏离物理合理范围。此时 L_spec 占主导——它强制模型收敛到物理边界内。随着训练的进行，L_scat 逐渐起主导作用——它精细调节散射一致性。

权重λ_scat=0.05 和 λ_spec=0.02 是经过平衡的。如果 λ_scat 太大（>0.1），模型会偏向于保守（feat_193 太接近 feat_266），丧失 3.5× 增益的效果。如果 λ_spec 太大（>0.05），边界附近会产生梯度不连续（ReLU 的硬边界），导致训练不稳定。0.05 和 0.02 的组合在实验中同时保证了物理合理性和 193nm 增益的有效性。

## 8. 三阶段训练管线设计

### 8.1 阶段一（无监督）：物理-informed CycleGAN

阶段一的训练数据是大量无标注暗场和明场图像，**以及少量 193nm 标定片图像**（~20 张配对标定片）。目标是让编码器和增强解码分支学会暗场→明场的域迁移，同时让 PISM 学会暗场→虚拟 193nm 的物理约束映射。

鉴别器配置：
- D_A：标准的 PatchGAN（ndf=64, n_layers=3），区分真实明场和生成的伪明场
- D_193：轻量 PatchGAN（ndf=32, n_layers=2），区分真实 193nm（来自标定片）和生成的伪 193nm

D_193 的设计必须轻量化，因为真实 193nm 训练数据极少。32 的 ndf 和 2 层结构将参数量限制在约 0.4M（D_A 的约 2.7M 参数的 15%）。这防止了鉴别器在少量训练数据上过拟合——如果鉴别器太强大，它会记住每一张真实 193nm 标定片，而不是学到一个通用的判别准则。

冻结策略：编码器和增强解码分支全量训练。PISM 的残差网络训练，物理部分（Scale Estimator、LUT）冻结。检测分支（266nm 和 193nm 头）全部冻结——阶段一无检测标注数据。

训练收敛的监控指标：
- FID < 30（生成明场 vs 真实明场）
- PISM 散射比偏差 < 15%（在标定片上）

### 8.2 阶段二（监督）：多角度数据微调

阶段二的训练数据是 ~50 张 266nm 缺陷标注图像 + ~10 组多角度套图（每组 0°+13 个角度）。目标是联合优化所有模块。

分级冻结策略：

```
Epoch 0-50:
  编码器 stages 1-2: 冻结（通用视觉特征不变）
  编码器 stages 3-4: 微调
  PISM 物理部分: 微调（允许 Scale Estimator 调优）
  PISM 残差网络: 训练（从配对标定片校准）
  266nm 检测头: 从零训练
  193nm 检测头: 从零训练
  ASG 调制+补全: 从多角度数据训练
  SGF: 从零训练

Epoch 51-100: 解冻 stage2
Epoch 101-200: 全部解冻，lr=1e-5
```

这里最关键的策略是 ASG 调制网络的训练方式。多角度套图提供了同一缺陷在 13 个角度下的真实图像。对于每个缺陷，我们在 0° 图像中标注了缺陷框，但没有在 13 个角度下都标注。损失的计算方式是：将 0° 特征通过 ASG 生成 13 个角度，在只有 0° 标注的情况下，我们计算「生成结果在 0° 位置附近的特征一致性」——即生成的某角度特征在语义上应与原始 0° 特征一致。这是跨角度自监督的一种形式。

小样本数据增强：

| 增强 | 作用 | 倍数 |
|------|------|------|
| Copy-Paste + 泊松融合 | 裁剪缺陷粘贴到其他图像 | 10× |
| Mosaic 拼接 | 4 张拼 1 张 | 4× |
| AugMix | 亮度/对比度/噪声随机变化 | 5× |

50 张标注 × 10 × 4 × 5 = 10000 等效样本，足够阶段二使用。

### 8.3 阶段三（知识蒸馏）：压缩到 ARM 可部署

阶段三的目标是将在阶段二训练得到的全功能教师模型（含真实 RepViT-M0.9 后约 8M 参数）蒸馏到一个学生模型（约 5M 参数）中。

除了标准的逻辑蒸馏（KL 散度）和特征蒸馏（MSE 对齐 stage2/3/4 特征）外，物理感知蒸馏新增三个目标：

1. **PISM 输出蒸馏**：MSE 对齐教师和学生 PISM 的输出特征。这确保学生 PISM 的散射增强行为与教师一致。
2. **193nm 分支逻辑蒸馏**：KL 散度对齐教师和学生在 193nm 检测头上的分类 logits。学生 193nm 分支（通道裁剪）需要学习教师的软标签分布。
3. **SGF 融合权重蒸馏**：L1 对齐教师和学生的 SGF 融合权重 w_193。如果学生的物理模块有偏差（比如增益偏低了），SGF 权重应该补偿。

蒸馏损失的权重配置为 0.05 + 0.05 + 0.03，远小于主蒸馏损失的 0.9。这是因为 PISM 和 SGF 主要是前向计算路径，蒸馏需求比检测头少——检测头需要学习复杂的类别边界，而物理模块只需要保持数值精度。

## 9. 模型压缩与 ARM 部署优化

### 9.1 从参数量到推理时间：ARM 部署的瓶颈分析

10M 参数的 FP32 模型权重约 40MB。ARM Cortex-A76 的 L2 缓存为 256-512KB——这意味着模型权重大部分时间不在缓存中，每次推理都需要从 DDR 内存读取。DDR 读取功耗约 10-20× 高于缓存访问，对于边缘设备（通常功耗上限 3-5W），这是不可忽视的约束。

INT8 量化将权重从 32 位压缩到 8 位，文件从 40MB 降到约 8MB（M0.9 方案）或约 3MB（M0.6 极致剪裁方案）。更重要的是，INT8 的卷积计算可以使用 ARM NEON 指令集（SIMD），比 FP32 快 2-4 倍。

### 9.2 重参数化结构融合

PISM 引入的物理模块不影响重参数化融合流程。PISM 本身不包含 RepVGGDW 类型的多分支结构——它的所有卷积都是标准 Conv-BN，可以直接融合为单 Conv。这得益于我们在设计 PISM 时就明确了「推理时单路」的约束。

RepViT 编码器的 fuse() 流程保持不变：
1. Conv2d_BN 融合：每个 Conv-BN 对 → 单 Conv（吸收 BN 参数）
2. RepVGGDW 融合：三分支 → 单 3×3 DWConv
3. Residual 融合：Identity 分支吸收到 Conv 权重
4. Dropout → Identity

融合后计算图中的 Conv2d 节点数减少 40-50%，ARM 推理速度提升 20-30%。

### 9.3 INT8 静态量化与敏感层处理

量化过程在融合后的模型上执行：

1. 校准：用 20 张代表性晶圆图像（包含各类缺陷和正常样本）收集激活值分布
2. 量化配置：`torch.quantization.get_default_qconfig('qnnpack')`，权重 per-channel，激活值 per-tensor
3. 敏感层跳过 FP32：编码器首层 patch_embed、增强输出层 final_conv、检测头分类/回归输出

敏感层在量化后的精度损失通常 < 1%（mAP 下降 < 0.5%）。如果需要更高的压缩比，可以执行量化感知训练（QAT）——在模拟 INT8 推理的约束下额外微调 5-10 epoch。

### 9.4 并行计算与 NEON 优化

PISM 前向在 ARM 上的计算包含以下几个步骤：Scale Estimator（2 个 3×3 卷积）、增益计算（常数乘法 + LUT 插值）、残差网络（2 个 1×1 卷积）。其中：
- 3×3 卷积：ARM NEON 的 `vmla.s8` 指令加速
- LUT 插值：由于 512 点 LUT 很小，可以预加载到 L1 缓存
- 1×1 残差卷积：GEMM 优化，使用 `sgemm_8x8` 内核

ASG 的 `torch.roll` 在 ARM 上是一个内存重映射操作——不涉及计算。13 个角度的 batch 拼接只需要在预处理阶段做一次内存拷贝。

### 9.5 JIT 导出与物理模块兼容性

JIT 导出是 ARM 部署的关键环节——它将 PyTorch 的动态计算图「冻结」为静态的 TorchScript 图，消除 Python 运行时依赖。PISM 和 ASG 的 JIT 兼容性在设计初期就被重点考虑：

PISM 的 JIT 兼容性保障：前向路径中没有任何 Python 原生操作（`.item()`、`.tolist()`、动态 `if/else`）。`F.grid_sample` 用于 LUT 插值是 PyTorch 标准算子，在 JIT trace 中完整保留。`self.register_buffer` 存储的 Mie LUT 在 JIT 图中变为常量张量，不占用运行时存储。

ASG 的 JIT 兼容性牺牲：`torch.roll` 的 `shifts` 参数接受 Python int，在 trace 时被固化。这意味着 trace 后的模型不能处理动态改变的角度数——这是可以接受的，因为部署时的角度集合是固定的。ASG 的 13 角度循环通过 `torch.jit.trace` 的 `unroll` 机制展开为 13 个固定的 roll 步骤，每个步骤的角度索引是硬编码常量。

已识别的兼容性问题：`torch.jit.trace` 默认 `strict=True` 时，对包含 dict 输出（PISM 和 SGF 都有 dict diagnostics）的模块会报 DeprecationWarning。当前使用 `strict=False` 绕过。需要在最终部署前用 `strict=True` 验证——如果某些操作无法 trace，需要在 forward 路径中用 TorchScript 兼容的写法替代。

### 9.6 内存占用分析与优化

推理时的峰值内存占用是 ARM 部署的第二个关键约束（目标 < 500MB）。分析各阶段内存需求：

1. **模型权重** (INT8)：~8MB（固定，不随 batch 变化）
2. **输入图像**：512×512×1×4 bytes = 1MB
3. **编码器中间特征**：F1~F4 四层 FP32 特征图约 56×128² + 112×64² + 224×32² + 448×16² ≈ 1.4M 像素 × 4 bytes = 5.6MB
4. **ASG 13 角度中间特征**：52 个 tensor × 各种尺寸，总计约 35MB
5. **Batch 13 检测**：batch=13 时各层特征扩大 13 倍，峰值约 70MB
6. **后处理缓冲区**：约 10MB

峰值约 130MB，远低于 500MB 上限。主要的内存优化点：ASG 的 52 个 tensor 不需要同时驻留在显存中——可以在计算 batch=13 检测前释放除当前角度外的其他角度特征。

### 9.7 实际基准测试结果与分析

在 CPU Python 环境下的实测基准（`deploy/benchmark.py`）：

| 配置 | 参数量 | 延迟 (CPU) | 说明 |
|------|--------|-----------|------|
| 无物理模块 | 1.46M | ~22ms | 原始单分支 |
| 双波长(无ASG) | 1.75M | ~44ms | PISM + 双检测 |
| **全功能(batch优化)** | **1.76M** | **~55ms (估)** | PISM + ASG + 双检测 |
| 全功能(未batch) | 1.76M | ~274ms | 修复前性能 |

未 batch 的 274ms 延迟中，约 260ms 来自 13 次独立的 193nm 检测循环。batch 优化后预期降至 55ms（ARM 上可进一步压缩 2-3×）。由于当前编码器是 placeholder（非 RepViT），实际部署时编码器参数从 1.2M 增至 5.1M，预计延迟增加约 10-15ms。

关键结论：**全功能配置的 ARM 推理时间约 55-70ms，满足 80ms 硬约束，但余量仅 12-18%。** 这意味着：
- 必须启用极坐标展开（再省 5-10ms）
- 必须完成 batch 优化（已修复）
- 建议在 ARM 开发板上做真实 benchmark 验证

### 10.3 三级降级预案

**一级降级（轻微精度损失）：**

将 ASG 角度数从 13 减少到 7（角度步长从 5° 改为 10°：15°/25°/35°/45°/55°/65°/75°）。推理时间节省约 50%，mAP 预计下降 1-2 个百分点。适用于 ARM 推理时间接近临界值或有功耗约束的场景。

**二级降级（双波长模式）：**

完全关闭 ASG 模块，退回双波长模式（仅 PISM 的 266nm→193nm 虚拟转换）。推理时间节省约 70%（从 55ms 降到约 22ms），但失去多角度融合带来的各向异性缺陷检出提升。适用于推理时间极度紧张（如低成本 ARM 芯片）或缺少多角度训练数据的场景。

**三级降级（原始单分支）：**

关闭所有物理增强模块，退回原始单分支检测架构。参数量从 ~5.7M 降至 ~5.1M，推理时间从 55ms 降至 22ms。此模式下不获得物理先验的任何增益，但模型仍然可以正常工作。适用于紧急降级场景（如硬件故障导致算力骤降）。

各级降级之间的切换是**运行时配置化的**，通过 `configs/physics_config.yaml` 单行修改即可，不需要重新训练或重新部署：

```yaml
physics:
  enabled: true
  asg:
    enabled: true        # 改为 false = 二级降级
    target_angles: [15, 30, 45, 60, 75]  # 从 13 减到 5
```

### 10.4 验收检查清单与通过标准

完整的四阶段验收流程覆盖从代码到产线的全链路：

**验收阶段一：代码审查**
审查人员逐项确认：所有模块 `forward()` 无 `if self.training`、`.item()`、`.tolist()` 等 JIT 不兼容操作；参数量在预算 ±10% 内；模块边界清晰无循环依赖；物理公式的数值正确性已核对。

**验收阶段二：功能正确性验证**
运行自动验证脚本确认：PyTorch 前向与 JIT 导出增强图 MaxDiff < 1e-5（FP32）和 < 1e-2（INT8）；结构融合后计算图中无 BatchNorm 节点；散射比偏差 < 10%；光谱角违例率 < 3%；双分支检测框 IoU > 0.85。

**验收阶段三：ARM 性能基准**
在目标 ARM Cortex-A76 开发板上执行：绑定单核 `taskset -c 0`，预热 10 次，统计 100 次推理取平均。全功能配置应 < 80ms，INT8 文件 < 8MB，峰值内存 < 500MB。任意一项不达标参考降级预案调整。

**验收阶段四：产线端到端验证**
使用 100 张产线标注图像（4 类缺陷各 > 20 张）执行完整检测流程，Python 端与 C++ 端逐像素逐框对比：增强图像素一致性 > 0.999，检测框类别一致性 100%，框 IoU > 0.99。**小缺陷召回率较无物理基线提升 > 5%** 是衡量物理模块核心价值的门槛指标。

### 10.5 风险应对矩阵

| 风险 | 触发条件 | 概率 | 影响 | 应对 |
|------|---------|------|------|------|
| INT8 量化后 mAP 下降 > 1% | 敏感层量化后精度骤降 | 中 | 高 | 敏感层保留 FP32；校准集覆盖全缺陷类；QAT 微调 5-10 epoch |
| ARM 推理达不到 80ms | 实际延迟 > 90ms | 低 | 高 | 角度数降 13→7；关 ASG；启用极坐标展开；换 M0.6 骨干 |
| 小缺陷召回率提升不达 5% | ASG+193nm 增益微弱 | 中 | 中 | 检查 PISM 增益是否被残差网络「抵消」；增大 L_scatt 权重 |
| 虚拟 193nm 与真实差异大 | 散射比偏差 > 15% | 中 | 中 | 增加配对标定片数量；增大残差网络容量（c//16→c//8） |
| 产线光源漂移后性能下降 | 运行 3 月+后 mAP 降 > 3% | 低 | 中 | 在线自适应校准每批次自动运行；季度增量微调 |
| ASG JIT 导出失败 | torch.roll 动态索引报错 | 低 | 高 | 改用 torch.jit.script 模式；预展开 13 个固定 roll 步骤 |

### 10.6 在线维护与模型持续更新

物理模块的一个关键优势是**可维护性**。PISM 的 Mie LUT 是基于理论计算的，不会因为数据分布偏移而失效——这是与纯数据驱动模型最本质的区别。当产线运行 6-12 个月后，光源老化导致的信号漂移主要影响解码器和检测头，而 PISM 的物理增益几乎不变。

维护策略：每月运行一次 `verify_physics.py` 检查散射比偏差趋势。如果偏差从 7% 缓慢上升到 12%，说明物理假设与真实物理过程的偏差在增大——这是残差网络需要重新校准的信号。重新校准仅需收集 5-10 对新的配对标定片，训练残差网络 5 epoch 即可恢复。

ASG 的维护更为简单——`torch.roll` 的几何变换不依赖任何参数，而散射调制网络学到的各向异性模式在物理上是稳定的。如果晶圆工艺发生变化（如从 SiC 切换到 GaN），需要更新的是检测头的分类能力，而非 ASG 的角度变换。
