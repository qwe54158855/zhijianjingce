## 1. 项目背景与三大核心难题

### 1.1 行业现状

半导体制造过程中，晶圆边缘缺陷检测是保证芯片良率的关键环节。SiC（碳化硅）、GaN（氮化镓）等第三代半导体材料广泛应用，带来三重新挑战。

### 1.2 三大核心难题

#### 难题一：国产光源暗场信噪比不足

**物理原理**：暗场检测通过倾斜入射照明使缺陷散射光进入探测光路。国产光源在功率稳定性、光谱纯度、光束均匀性三方面存在差距。

**量化数据**：
- 暗场 SNR：15-20dB（明场 30dB+）
- < 10 像素的微小缺陷（崩边起始裂纹 1-2μm、纳米级颗粒）被背景噪声淹没
- 传统图像增强方法（直方图均衡化、Retinex 分解）在低 SNR 下产生噪声放大效应

**应对方案**：学习型暗场→明场域迁移，让模型自主学习从低信噪比暗场到高对比度明场风格的映射，在增强缺陷对比度的同时抑制噪声放大。

#### 难题二：边缘设备 ARM 算力严重受限

**硬件约束**：
- 采用 ARM Cortex-A76 系列嵌入式平台（非 GPU 服务器）
- 算力 ≈ 2015 年旗舰手机 CPU 水平
- L2 Cache：256-512KB

**量化推导**：
- 12 寸晶圆（直径 300mm）以 512×512 窗口步进扫描
- 覆盖整片晶圆需要 2000-4000 张子图
- 总检测时间 < 3min → 单帧 < 80ms
- 普通 AI 模型在 ARM 上通常需要 200-500ms

**应对方案**：RepViT 轻量骨干 + 深度可分离卷积 + INT8 量化 + 重参数化融合，全部围绕 ARM 推理速度目标展开。

#### 难题三：第三代半导体样本极端稀缺

**数据现实**：
- 标注样本通常只有 30-50 张
- 标注需要半导体工艺专家逐像素标注，100-200 元/张
- 4 类缺陷平均每类仅 ~10 张
- SiC/GaN 工艺快速迭代，上个月的缺陷 ≠ 下个月的缺陷

**应对方案**：三阶段半监督训练（CycleGAN 无监督预训练 → 少量样本联合微调 → 知识蒸馏压缩），标注需求从 5000 张降到 50 张。

### 2.2 「一骨干双分支」设计原理

核心模式为「共享编码器 + 双任务解码分支」，基于多任务学习（MTL）和表征学习理论。

**优化目标**：
```
min θ_E,θ_e,θ_d  L_total = λ_e · L_e(D_e(E(x)), y_e) + λ_d · L_d(D_d(E(x)), y_d)
```

**三大优势**：

1. **参数效率**：图像低级特征（边缘、纹理、角点）对增强和检测通用，无需重复存储。总参数 ~7.5M，比独立部署两个模型（~7M+）减少约 30%

2. **计算效率**：只需一次编码器前向计算，两个分支共享中间特征图，节省近 40% 计算量

3. **表征协同**：增强任务梯度反向传播有助于编码器学到对光照变化、噪声模式鲁棒的特征，对检测任务泛化有正向促进作用。联合训练后检测精度比分别训练高 3-5 个 mAP 点

### 2.3 参数预算分配

| 组件 | 参数量 | 占比 | 说明 |
|------|--------|------|------|
| 共享编码器 | ~5.1M | 68% | RepViT-M0.9，精度-速度最佳平衡 |
| 增强解码分支 | ~1.2M | 16% | 深度可分离卷积 U 型，MSEFBlock×2 |
| 检测输出分支 | ~1.2M | 16% | C2f_MSD×2 + 轻量检测头（含 F2 小目标层） |
| **合计** | **~7.5M** | **100%** | 留 25% 余量 |

v2.0 新增光学物理模块约 0.31M 参数，总参数约 8M，余量 18.7%。

## 3. 共享编码器 —— RepViT 轻量骨干

### 3.1 RepViT 核心创新

RepViT（Rethinking Efficient Vision Transformer）由清华大学 THU-MIG 团队提出，结合 CNN 的高效部署特性和 Transformer 的全局建模能力。

#### （1）Token Mixer + Channel Mixer 双模块设计

每个 `RepViTBlock` 由 token_mixer（空间维度的特征交互）和 channel_mixer（通道维度的特征变换）组成，借鉴 Transformer 的 MHSA + FFN 结构，但全部使用卷积操作实现：

- **token_mixer**：深度可分离卷积（3×3 DWConv）做空间特征混合 + SE 注意力做通道重标定
- **channel_mixer**：倒残差结构 + 残差连接

```python
class RepViTBlock(nn.Module):
    def __init__(self, inp, hidden_dim, oup, kernel_size, stride, use_se, use_hs):
        # token_mixer: 空间特征混合（深度可分离卷积 + SE 注意力）
        # channel_mixer: 通道特征混合（倒残差结构 + 残差连接）
```

#### （2）RepVGGDW 重参数化结构（核心）

训练时为三分支并行，推理前融合为单路 3×3 深度卷积：

```python
class RepVGGDW(torch.nn.Module):
    def __init__(self, ed):
        # 训练时为三分支：
        # branch1: Conv2d_BN(3×3 DWConv)   — 主分支
        # branch2: Conv2d(1×1 DWConv)       — 辅助分支（pad 扩展为 3×3）
        # branch3: Identity (BN only)       — 恒等分支

    def forward(self, x):
        return self.bn((self.conv(x) + self.conv1(x)) + x)
```

融合后只有一个 3×3 DWConv，推理速度与普通 3×3 完全一致。

#### （3）Conv2d_BN 系统化融合

所有 Conv+BN 组合封装为 `Conv2d_BN` 类，提供统一的 `fuse()` 方法，吸收 BN 的均值和方差参数到 Conv 权重中。完整 fuse 流程：

```
Step 1: 遍历所有子模块，对每个 Conv2d_BN 调用 fuse()
         → Conv2d + BN → 单 Conv2d (absorb bias)
Step 2: 对每个 RepVGGDW 模块调用 fuse()
         → 3×3 DWConv + 1×1 DWConv(padded) + Identity → 3×3 DWConv
Step 3: 对每个 Residual 模块检查内置 Conv2d_BN 并融合 identity 分支
Step 4: 对 BN_Linear 模块调用 fuse() → BN + Linear → 单 Linear
Step 5: nn.Dropout → nn.Identity
```

融合后 Conv2d 节点减少 40-50%。

### 3.2 从分类到多尺度编码器的改造

| 改造项 | 原始 RepViT | 本项目改造 |
|--------|------------|-----------|
| 输入通道 | RGB 3 通道 | 单通道灰度（in_chans=1）或多波长（in_chans=4） |
| 输出方式 | 最后一层 → 分类头 | 4 层中间特征输出 F1/F2/F3/F4 |
| 分类头 | 全局平均池化 + Linear | 删除，代之以多尺度特征接口 |

**4 层多尺度特征**：

| 层级 | Stride | 分辨率 | 通道数 | 供给分支 |
|------|--------|--------|--------|---------|
| F1 | 4 | 128×128 | 56 | 增强分支（纹理细节） |
| F2 | 8 | 64×64 | 112 | 增强+小目标检测 |
| F3 | 16 | 32×32 | 224 | 检测分支（中层语义） |
| F4 | 32 | 16×16 | 448 | 检测分支（高层语义） |

### 3.3 为什么选择 RepViT？

| 候选网络 | 参数 | ImageNet Top-1 | ARM 延迟 | 重参数化 | JIT 兼容 |
|---------|------|---------------|---------|---------|---------|
| **RepViT-M0.9** | **5.1M** | **78.7%** | **~0.9ms** | **✅ 原生** | **✅** |
| MobileNetV3-L | 5.4M | 75.2% | ~12ms | ❌ | ✅ |
| EfficientNet-B0 | 5.3M | 77.1% | ~22ms | ❌ | 需改造 |
| ShuffleNetV2 | ~5M | 72.6% | ~8ms | ❌ | ✅ |

选择 RepViT 的核心理由：
1. **原生重参数化** — 融合流程是设计时就考虑好的系统特性，融合后的单路 3×3 DWConv 对 ARM NEON 指令集高度友好
2. **JIT 兼容性经过验证** — forward 无动态控制流、无 Python 原生操作、无自定义算子
3. **社区活跃度高**（1.7k Stars），FALCO-WAFER 已有集成经验

## 4. 增强解码分支 —— 暗场→明场域迁移与降噪

### 4.1 暗场噪声分析

| 噪声类型 | 来源 | 特性 |
|---------|------|------|
| 散粒噪声（Shot Noise） | 光子计数泊松分布 | 随机分布亮点，强度与信号平方根关系 |
| 读出噪声（Readout Noise） | CMOS 读出电路 | 低照度下占比增大 |
| 固定模式噪声（FPN） | 像素响应不一致 | 固定列/行条纹，国产传感器更突出 |

### 4.2 增强目标建模

将增强过程建模为从暗场域 X 到明场域 Y 的映射 G: X → Y，满足四个约束：
1. **内容保持**：保留结构信息（晶圆边缘、缺陷轮廓），不产生伪影
2. **噪声抑制**：有效滤除噪声，不过度平滑缺陷区域
3. **对比度提升**：缺陷与背景对比度显著提高
4. **风格迁移**：视觉风格与明场一致（光照均匀、背景平滑）

### 4.3 解码分支架构

```
输入特征: F1 (128×128×56) F2 (64×64×112) F3 (32×32×224)

Step 1: 通道对齐（1×1 卷积压缩）
  F1_proj = Conv1×1(56→32)   → 128×128×32
  F2_proj = Conv1×1(112→48)  → 64×64×48
  F3_proj = Conv1×1(224→48)  → 32×32×48

Step 2: 自底向上融合
  x = F3_proj                 → 32×32×48
  x = UpSample(×2) + F2_proj  → 64×64×96
  x = MSEFBlock(x)            → 64×64×48
  x = UpSample(×2) + F1_proj  → 128×128×80
  x = MSEFBlock(x)            → 128×128×48

Step 3: 残差降噪
  x = Denoiser(x)             → 128×128×48
  x = Conv3×3(48→1)          → 128×128×1
  x = Sigmoid                 → [0,1]

Step 4: 上采样到原图
  enhanced = Upsample(×4) + Conv → 512×512×1
```

### 4.4 MSEFBlock 模块

```
MSEFBlock:
  LayerNorm → DWConv(3×3) → SE 注意力 → 逐元素乘积融合 → 残差连接
```

设计理念：「空间-通道解耦注意力」
- `depthwise_conv`（空间）：每个通道独立做 3×3 卷积，参数量为普通 3×3 Conv 的 1/filters
- `se_attn`（通道）：全局平均池化 → 两个 FC → Tanh 激活 → 通道权重
- `x1 * x2`：x1 告诉模型「哪里有结构」，x2 告诉模型「哪些通道可信赖」

**为什么用 Tanh 而非 Sigmoid？** LYT-Net 实验表明 Tanh（输出范围 [-1,1]）比 Sigmoid（[0,1]）在低光照增强任务中梯度流动更好。

### 4.5 Denoiser 降噪模块

#### U 型结构

```
输入 x (128×128×48)
  ├── Conv3×3(48→48) stride=1   → 128×128×48
  ├── Conv3×3(48→48) stride=2   → 64×64×48       ↓下采样
  ├── Conv3×3(48→48) stride=2   → 32×32×48       ↓下采样
  ├── 瓶颈注意力（二选一）:       → 32×32×48
  │   ├── [精度优先] MHSA(embed=48, heads=4)
  │   └── [速度优先] DWConv7×7 + SE
  ├── UpSample(×2) + skip       → 64×64×96       ↑上采样
  ├── UpSample(×2) + skip       → 128×128×96     ↑上采样
  ├── Conv3×3(96→48)            → 128×128×48
  └── Conv3×3(48→1) + Tanh      → 128×128×1     输出残差
```

#### 区分噪声和微小缺陷的四种机制

**① 全局上下文做背景参考（瓶颈注意力）**
位于 32×32 瓶颈处的注意力模块使每个位置都能参考远处上下文。孤立亮点找不到远处相关纹理模式 → 判为噪声抑制；按晶圆纹理方向连续延伸的信号 → 判为缺陷保留。

**② 残差学习（宁漏勿杀）**
Denoiser 输出残差信号（Tanh 范围 [-1,1]），最终输出 `x + Denoiser(x)`。初始时残差接近 0，网络学到的第一件事是「不确定的就不动」。默认保守策略防止误把缺陷当噪声。

**③ 多尺度交叉验证**
从 F1(128×128) 和 F2(64×64) 两个尺度接入特征。小缺陷在两个尺度上都有响应但形态不同；纯噪声通常只在一个尺度上有响应。

**④ LPIPS 感知损失监督**
训练时损失包含 LPIPS 感知损失——不是比「像素对不对」，而是比「看起来像不像明场」。如果 Denoiser 把缺陷也磨平了，LPIPS 损失会很高（因为明场中缺陷应清晰可见）。

#### 瓶颈注意力模式选择

| 模式 | 结构 | 参数量 | 推理速度 | 适用场景 |
|------|------|--------|---------|---------|
| MHSA（精度优先） | embed=48, heads=4 | ~10K | 慢（ARM 效率低） | 训练/高精度 |
| DWConv7×7+SE（速度优先） | 7×7 DWConv + SE | ~5K | 2-3× 快于 MHSA | ARM 部署默认 |

通过 `denoiser_mode: "mhsa" | "dwconv"` 一键切换。

### 4.6 增强输出的后处理

Sigmoid 激活后有时出现对比度压缩，在 C++ 后处理中加入可选自动对比度拉伸：

```python
enhanced = enhanced * 2.0 - 1.0
enhanced = (enhanced - enhanced.min()) / (enhanced.max() - enhanced.min() + 1e-8)
```

JIT 图中只保留 Sigmoid 输出，体现「JIT 图做核心推理，外围处理由宿主代码完成」原则。

## 5. 检测输出分支 —— Anchor-free 缺陷检测

### 5.1 为什么选择 Anchor-free？

晶圆缺陷尺寸和形状变化极大——从 < 10 像素的微小颗粒到数百像素的划痕。Anchor-based 方法需要预先设计锚框尺寸，难以覆盖这种极端长尾分布。Anchor-free 直接预测每个空间位置到目标框的四边距离。

### 5.2 检测头架构

**针对晶圆场景的四项精简优化**：

1. **去掉独立的 Neck 结构（FPN/PAN）**：F2/F3/F4 已形成天然特征金字塔，去掉 FPN 后精度损失 < 0.5 mAP，参数量减少约 2M

2. **C2f_MSD 替代标准 C2f**：使用 DynamicIncMixerBlock 多尺度深度可分离模块，同参数量下有效感受野更大

3. **新增 F2 小目标层**：三层多尺度 F2(stride=8)/F3(stride=16)/F4(stride=32)。10px 缺陷在 stride=8 图上对应 1.25px，使小缺陷具有可分辨特征响应

4. **简化回归头**：去掉 DFL（Distribution Focal Loss），回归参数量从 4×16=64 降到 4，减少 93%

```python
class WaferDetectHead(nn.Module):
    def __init__(self, num_classes=4, ch=(112, 224, 448)):
        self.msd_f2 = C2f_MSD(ch[0], ch[0]//2)   # 112→56
        self.msd_f3 = C2f_MSD(ch[1], ch[1]//2)   # 224→112
        self.msd_f4 = C2f_MSD(ch[2], ch[2]//2)   # 448→224
        # 分类头 + 回归头（简化，无 DFL）
        self.cls_f2 = nn.Conv2d(56, num_classes, 1)
        self.reg_f2 = nn.Conv2d(56, 4, 1)         # 直接 xywh
```

### 5.3 缺陷类别定义

| ID | 英文名称 | 中文名称 | 特点 | 典型尺寸 | 检测难度 |
|----|---------|---------|------|---------|---------|
| 0 | Chipping | 崩边 | 边缘材料碎裂，V 形或弧形缺口 | 10-200μm | 中 |
| 1 | Particle | 颗粒 | 表面附着异物颗粒，近球形 | 1-50μm | 高（小目标） |
| 2 | Scratch | 划痕 | 细长线性损伤，各向异性最明显 | 长度 50-500μm | 低（高通量） |
| 3 | Dislocation | 位错 | 晶格缺陷，暗色线条或团簇 | 5-100μm | 高（对比度低） |

### 5.4 检测后处理流水线

```
Step 1: 解码预测值（三层多尺度）
  F2: cls_f2(B×4×H2×W2), reg_f2(B×4×H2×W2)   # stride=8
  F3: cls_f3(B×4×H3×W3), reg_f3(B×4×H3×W3)   # stride=16
  F4: cls_f4(B×4×H4×W4), reg_f4(B×4×H4×W4)   # stride=32

Step 2: 置信度筛选 → 保留 conf.max(dim=1) > 0.25

Step 3: 所有尺度合并 → 对每类缺陷独立做 NMS（IoU 阈值 0.45）

Step 4: 输出 [N, 6]: [x_center, y_center, w, h, class_id, confidence]
```

### 6.2 散射理论基础

#### （1）Rayleigh 散射（小尺寸缺陷）

适用条件：d << λ（缺陷尺寸 < 42nm）

```
散射强度 I_scat ∝ (1/λ⁴) × |(n²-1)/(n²+2)|² × d⁶
```

三层物理含义：
- **与波长四次方成反比** — 266nm→193nm：`(266/193)⁴ ≈ 3.5`
- **与缺陷体积平方成正比**(d⁶) — 10nm 颗粒散射强度是 5nm 的 64 倍
- **依赖折射率对比度** — SiC(n≈2.6) 在空气(n=1.0)中对比度因子约 0.67，显著高于非晶颗粒(n≈1.5)的 0.04

**核心公式**：`I(193nm)/I(266nm) = (266/193)⁴ ≈ 3.5`——驱动整个虚拟波长转换方案的核心物理动力。

#### （2）Mie 散射（大尺寸缺陷）

适用条件：d ~ λ 或 d > λ

Maxwell 方程组边值问题的精确解析解，散射效率：
```
Q_sca = (2/x²) × Σ(2n+1) × (|aₙ|² + |bₙ|²)
```

其中 aₙ、bₙ 为 Mie 散射系数，由尺寸参数 x = πd/λ 和复折射率 m 决定。

**工程实现**：Bohren-Huffman 算法预计算 512 点 Mie LUT（SiC n=2.6+0.1i@266nm / n=2.8+0.15i@193nm），`F.grid_sample` 可微插值。

#### （3）Rayleigh-Mie 高斯加权混合模型

```
gain(s) = w_r(s) × 3.5 + [1 - w_r(s)] × Mie_LUT(s)
```

- s ∈ [0,1]：每像素散射尺度（s→0 小缺陷，s→1 大缺陷）
- w_r(s) = exp(-s²/2σ²)，σ=0.3：高斯过渡权重
- s→0：Rayleigh 主导，gain ≈ 3.5×
- s→1：Mie 主导，gain ≈ 1.2-2.0×
- 光谱角约束：clamp[1.0, 8.0]

#### （4）各向异性散射

| 缺陷类型 | 散射特性 | 垂直/平行差异 |
|---------|---------|-------------|
| 颗粒（Particle） | 各向同性，近似球形对称 | ~1× |
| 划痕（Scratch） | 高度各向异性 | 10-20× |
| 崩边（Chipping） | 介于两者之间 | 3-5× |

单角度检测将所有信息压缩为标量，多角度检测得到向量——像光谱一样用于缺陷类型识别。

### 6.3 极坐标变换与方位角物理

#### 为什么是极坐标？

- 晶圆是圆形薄片，缺陷集中在边缘环形区域
- 环形区域仅占 512×512 图像面积约 35-40%
- 60% 以上像素是无效背景

**极坐标展开**：262K 像素(512×512) → 65K 像素(512×128)，计算量减少 60%，ARM 推理 +30%。

#### 关键同构：方位角旋转 = 特征图水平平移

在极坐标空间中，宽度方向 = 360° 方位角。旋转 Δθ 在特征图上等价于水平平移 Δx：

```
Δx = round(Δθ × 512 / 360)  # 像素偏移
```

实现：`torch.roll(feature, shifts=shift, dims=-1)`——**零参数、零计算成本、O(1) 时间复杂度**。

**13 个目标角度**：15°, 20°, 25°, 30°, 35°, 40°, 45°, 50°, 55°, 60°, 65°, 70°, 75°（步长 5°）。

### 6.4 PISM：可微散射物理模块

**位置**：共享编码器之后、检测分支之前  
**设计哲学**：「物理 90 + 学习 10」——物理公式提供 90% 主信号，可学习残差补偿 10% 近似误差

#### 三步计算流程

**① 局部尺度估计（Scale Estimator）**

每像素估计散射尺度 s ∈ [0,1]（从特征图局部响应模式推断）：
- s → 0：小缺陷（Rayleigh 区，尖锐响应）
- s → 1：大缺陷（Mie 区，扩散响应）

结构：`Conv2d(C→16, 3×3) → BN → ReLU → Conv2d(16→1, 3×3) → BN → Sigmoid`

每层 ~2.5K 参数，4 层合计 ~10K。每特征层有独立 Scale Estimator（不同分辨率对不同尺度敏感）。

**② Rayleigh-Mie 混合增益计算**

```python
w_r = torch.exp(-s**2 / (2 * sigma**2))  # sigma=0.3
combined_gain = w_r * rayleigh_ratio + (1 - w_r) * mie_gain
combined_gain = combined_gain.clamp(1.0, 8.0)  # 光谱角约束
```

- Rayleigh 部分：常数 (266/193)⁴ ≈ 3.5（**物理常数，不靠学**）
- Mie 部分：512 点 LUT 预计算，`F.grid_sample` 可微双线性插值
- Mie LUT 冻结（`register_buffer`），不参与训练

**③ 物理增强 + 残差补偿**

```python
feat_193_phys = feat_266 * combined_gain      # 物理驱动增强
feat_193 = feat_193_phys + 0.1 * residual_net(feat_266)  # 残差微调
```

残差缩放 0.1 确保初始状态物理驱动完全主导，残差仅补偿 ~10% 非理想效应（非球状粒子、多次散射、折射率色散）。

#### 给负责人的类比

> 用**公式保证下限**，用**学习逼近上限**。模型不需要自己去「领悟」短波长信号更强——我们把 Rayleigh 公式写进网络里，它只需要学一件事：当前像素的散射尺度 s 是多少。就像给质检员一本物理公式手册，他只需选「这个缺陷算小还是大」，剩下的增益由物理公式自动算好。

#### 有物理先验 vs 纯数据驱动的区别

| 维度 | 纯数据驱动（GAN） | PISM 物理先验 |
|------|-----------------|--------------|
| 遇未见缺陷类型 | 可能乱猜（违反物理） | 物理公式保底，不会太离谱 |
| 训练数据量 | 需要大量标注 | 90% 信号来自物理，只需少量数据校准 10% |
| 可解释性 | 黑箱 | 输出 gain_map/scale_map 诊断，回答「为什么增强 3 倍」 |
| 参数效率 | 需大网络自己摸索 | 仅 0.16M 参数 |

### 6.5 ASG：多方位角度散射生成器

**一句话**：把晶圆从圆形展开为长条形，旋转 = 水平滑动（torch.roll 零成本实现）。

#### 三层结构

```
ASG 单角度前向:

Step 1: 几何重投影（0 参数, 零计算）
  shifted = torch.roll(feat, shift=θ×512/360, dims=-1)
  ↑ 物理上精确的方位角旋转，O(1) 时间复杂度

Step 2: 散射强度调制（~13K 参数）
  feat = shifted × (1 + 0.3 × Tanh(角度编码 + 散射尺度))
  ↑ 学习各向异性缺陷的散射强度随角度变化

Step 3: 轻量补全（~10K 参数, 仅 F1 层）
  feat = feat + 0.1 × completion_net(feat)
  ↑ 补偿大角度下晶圆边缘曲面遮挡
```

| 指标 | 值 |
|------|-----|
| 13 角度总参数量 | ~23K |
| 生成速度 | < 0.1ms |
| 对比 GAN/NeRF | 节省 1000+ 倍参数和推理时间 |
| 小缺陷召回率提升 | +5-8% |
| 推理可降级 | 13→7 角度（运行时切换） |

### 6.6 双光路并行检测

#### 为什么不能共享权重？

两个波长的散射特性差异太大：共享参数会让网络只能找到一个「两头都不靠」的折中。

**266nm 分支**（全通道基准）：
- 衍射极限 133nm，更接近光路真实物理值 → **定位精确**
- F2(112→56), F3(224→112), F4(448→224) 全通道
- ~0.25M 参数，从阶段一预训练编码器受益

**虚拟 193nm 分支**（通道裁剪）：
- Rayleigh 3.5× 增益 → **召回率高**（小缺陷信号显著增强）
- F2(112→56)**保留全通道**（3.5× 对小缺陷最有利），F3(224→64)/F4(448→128) 通道裁剪 43%
- ~0.12M 参数，从 266nm 分支复制前 k 通道权重初始化

#### F2 全通道保留的物理依据

10px 缺陷在 stride=8 特征图上仅 1.25px。266nm 下 SNR 过低，1.25px 的信息非常勉强；193nm 的 3.5× 增益使信号不再被噪声淹没——需要全通道容量编码高 SNR 信息。

#### 批处理 13 角度的性能优化

```python
# ❌ 修复前：13 次独立前向 → ~260ms
for i in range(13):
    angle_feats = all_angle_feats[i*4+1 : i*4+4]
    output = detect_193(angle_feats)

# ✅ 修复后：batch=13 一次前向 → ~4ms
feats_batch = [torch.cat([all_angle_feats[i*4+j] for i in range(13)], dim=0) for j in [1,2,3]]
batch_output = detect_193(feats_batch)
```

#### 跨分支梯度平衡

前 50 epoch 启用动态损失缩放：`ratio = L_193/(L_266 + 1e-6)`
- ratio > 2.0：L_193 缩放为 L_193/ratio
- ratio < 0.5：L_266 缩放为 L_266×ratio

50 epoch 后取消缩放，两个分支自由竞争。

#### 联合 NMS 的软投票效应

```
266nm 分支：定位精度高（真实衍射极限）
193nm 分支：召回率高（3.5× 增益）
联合 NMS：用高灵敏度发现目标，用高精度定位目标
```

### 6.7 两级融合策略：角度注意力 + SGF

#### 角度注意力融合（13 → 1）

```python
W_final = 0.7 × W_physics + 0.3 × W_learned
```

- **W_physics**：ASG 自评估置信度归一化——各向同性(13 角度等权)，各向异性(物理权重自动聚焦)
- **W_learned**：Conv2d(13C→64)→ReLU→Conv2d(64→13)→Softmax——补偿跨角度异常模式

#### SGF 散射引导融合（波长融合）

```
w_193 = Sigmoid(Conv2D(gain_map || scale_map))
fused = w_266 × pred_266 + w_193 × pred_193
```

- gain_map ↑ → w_193 ↑（高增益区域信赖 193nm）
- scale_map s→0(Rayleigh) → w_193 趋近 0.9
- scale_map s→1(Mie) → w_193 趋近 0.4-0.5

结构：4 层轻量卷积网络 Conv2d(2→8)→BN→ReLU→Conv2d(8→1)→BN→Sigmoid，~8K 参数。

#### 完整融合流程

```
Step 1: ASG 生成 13 角度的 193nm 特征
Step 2: 193nm 检测头在 batch=13 上预测
Step 3: 角度注意力融合 → 13-in-1 融合预测
Step 4: SGF 接收增益图 → 生成融合权重 w_193
Step 5: fused = w_266 × pred_266 + w_193 × pred_193
```

**修正细节**：增益图需随角度 roll 以对齐特征图坐标。

### 6.8 物理约束损失函数

#### 散射一致性损失 L_scat

```
L_scat = Σ ||feat_266 - PISM(feat_193)||²
```

约束虚拟 193nm 特征通过 PISM 反算回散射域时与原始 266nm 响应一致。
- 权重 λ_scat = 0.05（阶段一、二）
- `torch.no_grad()` 双重保险防止 PISM 梯度传播到 feat_193

#### 光谱角损失 L_spec

```
L_spec = mean[ReLU(1.0 - ratio) + ReLU(ratio - 8.0)]
其中 ratio = |feat_193| / |feat_266|
```

- **下界 1.0**：193nm 不应弱于 266nm（违反 Rayleigh 律）
- **上界 8.0**：留 2× 裕量给共振增强效应
- 权重 λ_spec = 0.02（阶段一、二）

#### 三阶段损失权重配置

| 损失项 | 阶段一（无监督） | 阶段二（联合微调） | 阶段三（蒸馏） |
|--------|----------------|------------------|--------------|
| L1_cycle + GAN + LPIPS | 10.0 + 1.0 + 0.06 | — | — |
| L_scat + L_spec | 0.05 + 0.02 | 0.05 + 0.02 | — |
| L_det(266+193+fused) | — | 1.0+1.0+0.5 | — |
| KL 蒸馏 + MSE 特征 | — | — | 0.9 + 0.1 |

### 7.2 阶段一：CycleGAN 无监督域迁移预训练

**核心机制**：循环一致性损失（Cycle Consistency Loss）

```
L_cycle = E[||G_B(E(x), x) - x||₁] + E[||G_A(E(y), y) - y||₁]
```

- G_A = 共享编码器 + 增强解码（暗→明）
- G_B = 共享编码器 + 轻量反向解码（明→暗，仅阶段一存在）
- 鉴别器：PatchGAN（NLayerDiscriminator），对局部纹理真实性做判断

**损失组合**：

| 损失项 | 权重 | 设计意图 |
|--------|------|---------|
| L1_cycle | 10.0 | 循环一致性（主损失），L1 对离群值更鲁棒 |
| L1_identity | 0.5 | 防止过度着色，约束域迁移只在必要时改变 |
| GAN_loss | 1.0 | LSGAN 最小二乘损失，梯度更平滑 |
| LPIPS | 0.06 | 感知质量度量，基于预训练网络的深度特征空间 |

**训练参数**：Adam(β₁=0.5, β₂=0.999), lr=1e-4, batch=16, 512×512, 100 epoch, 前 50 fix 后 50 decay。

### 7.3 阶段二：少量样本联合微调

**分级冻结（Progressive Unfreezing）**：

```
Epoch 0-50: 冻结底层(stages 1-2)，训练高层 + 检测头
Epoch 51-100: 解冻 stage2
Epoch 101-200: 全部解冻，lr=1e-5 精细调整
```

**数据增强组合**：

| 增强 | 倍数 | 效果 |
|------|------|------|
| Copy-Paste + 泊松融合 | 10× | 裁剪缺陷粘贴到正常晶圆，边界平滑 |
| Mosaic | 4× | 4 张拼 1 张，迫使学习任意位置检测 |
| MixUp | ∞ | 两图按比例混合，平滑决策边界 |
| AugMix | 5× | 多种随机增强混合，提高光照鲁棒性 |

50 × 10 × 4 × 5 = **10000+ 等效样本**。

### 7.4 阶段三：知识蒸馏压缩

## 8. 模型压缩与 TorchScript JIT 导出

### 8.1 重参数化结构融合

融合顺序（有依赖关系）：
```
Step 1: Conv2d_BN 融合（必须是第一步）
Step 2: RepVGGDW 融合（依赖 Step 1 结果）
Step 3: Residual 融合（Identity 分支吸收）
Step 4: Dropout → Identity
```

验证方法：融合前后 `torch.jit.trace` 追踪计算图，Conv2d 节点数应减少 40-50%。

### 8.2 INT8 静态量化

- 校准：20 张代表性晶圆图像（缺陷+正常混合），收集激活值分布
- 量化配置：权重 per-channel，激活 per-tensor（`qnnpack` 后端）
- 敏感层跳过 FP32：编码器首层 patch_embed、增强输出层 final_conv、检测头分类/回归输出
- 量化后精度：增强 MaxDiff < 1e-2，检测 mAP 下降 < 0.5%

### 8.3 TorchScript 追踪导出

```python
model.eval()
model = fuse_all(model)
model = quantize_model(model)  # 可选
dummy_input = torch.randn(1, 1, 512, 512)
traced = torch.jit.trace(model, dummy_input)
traced.save('wafer_detector_jit.pt')
```

**JIT 校验标准**：
```
✅ FP32 JIT: 增强图与 PyTorch 输出 MaxDiff < 1e-5
✅ INT8 JIT: 增强图 MaxDiff < 1e-2
✅ INT8 JIT: 检测类别一致性 > 99%
✅ JIT 模型文件大小: INT8 < 8MB (M0.9) / < 3MB (M0.6)
```

## 10. 晶圆场景专属优化工程

### 10.1 极坐标边缘展开

**背景**：缺陷几乎全部位于晶圆边缘环形区域（仅占 512×512 图像约 35-40%）

**方法**：
```
Step 1: 霍夫圆检测确定圆心（后备：图像中心）
Step 2: cv::warpPolar 展开为 512×128 矩形
Step 3: 在展开图上推理（512×128×1 而非 512×512）
Step 4: 检测结果反向映射（极坐标→笛卡尔坐标）
```

**效果**：像素减少 75%，计算量减少 60%，ARM 推理 +30%。

**注意事项**：对晶圆中心区域（位错团簇）不适用，可通过 `enable_polar` 开关关闭。

### 10.2 在线自适应校准

**问题**：设备运行导致光源亮度衰减、传感器灵敏度漂移 → 模型输入分布偏移

**流程**：
```
设备启动 → 收集 5-10 张正常晶圆 → 计算灰度统计量
→ 对比训练时基准 → 计算偏移量 → 调整归一化参数
→ 保存在 inference_config.json → 下次启动自动加载
```

**效果**：6 个月连续运行后，定期校准模型误检率仅上升 2%，未校准上升 12%。

### 10.3 参数化配置系统

通过外部 JSON 配置文件适配不同工艺条件（无需重新训练）：
- **检测灵敏度调节**：每类缺陷独立 sensitivity 参数，快速抽检模式提高召回/精密检测降低误报
- **晶圆尺寸适配**：4/6/8/12 英寸自动适配

## 12. 约束限制与扩展性

### 12.1 已知约束

| 约束 | 说明 | 应对 |
|------|------|------|
| 固定 512×512 输入尺寸 | JIT trace 机制限制 | 预处理 Resize + 后处理坐标映射 |
| 不适用视频流实时检测 | ~12 FPS，受机械平台移动限制 | 优化目标是总耗时而非帧率 |
| 极坐标不适用晶圆中心 | 对位错团簇等中心区域缺陷无效 | 关闭 `enable_polar` 开关回退全图 |
| INT8 精度边界 | 极端场景（过曝/欠曝/校准集不完整）可能 > 2% | per-channel 量化/跳过敏感层/QAT微调 |

### 12.2 三维扩展接口

**① 缺陷类别扩展**：修改 `num_classes` 参数 → 分类卷积输出通道自动调整 → 10% 标注数据 20 epoch 微调

**② 输入通道扩展**：`in_chans=4` 时自动插入 `MultiWavelengthFusion` 通道注意力模块（自适应融合 266nm+532nm 多波长输入）

**③ 跨工艺适配**：JSON 配置文件修改晶圆尺寸/检测 ROI/缺陷灵敏度/输出格式

### 12.3 降级预案（运行时配置化）

> **文档结语**  
> 本方案的核心设计哲学可以概括为「三个共享、两个代偿」：
> - **共享编码器**：一次提取特征，两个分支共用，省参数省计算
> - **共享物理先验**：PISM 将物理公式编码进网络，用 0.16M 参数换来 3.5× Rayleigh 散射增益
> - **共享几何先验**：极坐标 + torch.roll 零成本生成 13 个视角
> - **用算法代偿硬件**：不买昂贵的 193nm 光源，AI 模拟效果
> - **用半监督代偿数据**：50 张标注 + 大量无标注数据 = 上万个有效样本
>
> 全链路自主可控 · 不依赖进口光学硬件 · 算法全栈自主
