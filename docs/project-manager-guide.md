# 晶圆缺陷检测项目 — 负责人导读

> 写给非技术背景的项目负责人。用大白话解释每一项技术解决了什么实际产线问题。

---

## 一句话概括

**我们正在造一个「晶圆缺陷检测专用 AI 大脑」——只有指甲盖大小，装在一颗几十块钱的 ARM 芯片上，能在 3 分钟内扫完一整片 12 寸晶圆（相当于同时看 4000 张照片，每张只要 0.08 秒），并且边看边把照片修成高清、同时圈出缺陷的位置和类型。**

---

## 三个要命的现实问题

### 问题 1：国产光源暗场 SNR 不足

**背景**：晶圆暗场检测中，光线从侧面斜照到晶圆表面，正常情况下没有光进入镜头，只有缺陷才会把光散射进去。

**痛点**：国产光源在功率稳定性、光谱纯度、光束均匀性三方面与进口存在差距。暗场图像的信噪比低——噪声能量可能达到信号的 10% 以上。

**直接影响**：小于 10 像素的微小缺陷直接淹没在背景噪声里。

**我们的思路**：不是用固定公式去「修图」，而是让 AI 自己学——给它看大量暗场和明场照片，让它自己领悟什么样的暗场图应该被处理成什么样子。基于学习的增强方法可以区分「噪声」和「缺陷」。

### 问题 2：ARM 算力严重受限

**背景**：晶圆检测设备为控制成本，通常采用 ARM 嵌入式平台，而不是 GPU 服务器。大约相当于 2015 年旗舰手机的 CPU 水平。

**量化**：12 寸晶圆需要 2000-4000 张子图，总时间控制在 3 分钟内。倒推下来每张图处理不能超过 80 毫秒。而普通 AI 模型在 ARM 上跑一帧通常要 200-500 毫秒。

> 相当于要求你在智能手表上用《原神》保持 60 帧。

**我们的对策**：整个项目围绕「在 ARM 上跑进 80 毫秒」这个核心约束展开。

### 问题 3：第三代半导体缺陷样本极端稀缺

**背景**：SiC 和 GaN 的制造工艺还在快速迭代，每批次的缺陷形态都在变化。一个典型项目可用的标注样本只有 30-50 张。

**直接后果**：用 30-50 张图直接训练大型 AI 网络，模型会把这些图「背下来」。换一张新图就不认识了——这就是过拟合。

> 让一个学生只看 3 道例题就去参加高考。

**我们的对策**：三步走——① 先看大量不需要标注的图，自己摸索规律 ② 再用少量标注样本做精细调整 ③ 最后把大模型压缩成小模型。三步把「需要 5000 张标注」降到了「只需要 50 张标注」。

## 总体架构

```
暗场原图 → 共享编码器(通用眼睛) → 增强分支(修图) + 检测分支(找缺陷)
         → 明场增强图 + 缺陷列表
         → TorchScript JIT 单文件 → ARM 芯片实时推理
```

**核心创新**：增强和检测共用同一个「眼睛」（共享编码器），这比两套独立模型省 40% 计算量，而且联合训练后检测精度反升 3-5 个点。

### v2.0 重大升级：光学物理约束注入

| 模块 | 大白话解释 |
|------|-----------|
| PISM | 给 AI 一本物理公式手册，告诉它「短波长看小缺陷更清楚」这个物理规律 |
| ASG | 把晶圆展开成长条，旋转晶圆 = 水平滑动图片——零成本模拟 13 个角度看缺陷 |
| 双光路 | 一个分身专注「定位准」，一个分身专注「召回高」，两个配合 |
| SGF | 自动判断该信哪个分身的判断结果 |

**不花 50 万买进口 193nm 光源，用 AI 模拟。**

## 验收标准快速版

### P0 硬约束（不满足就不通过）
- 模型 < 1000 万参数
- 文件 < 8MB（跟一张高清 JPG 差不多）
- ARM 单核 < 0.08 秒/张
- 12 寸晶圆总耗时 < 3 分钟

### P1 软约束
- 检测精度 > 80%
- 增强质量 PSNR > 25dB（人眼看不出区别）
- 蒸馏后精度损失 < 1%

## 风险快速表

| 风险 | 可能性 | 怎么办 |
|------|--------|--------|
| 量化后精度下降 | 中 | 敏感部分保留高精度；额外校准 5-10 轮 |
| ARM 跑不进 80ms | 低 | 减少角度数；关掉部分功能；换更小模型 |
| 小缺陷检出不够 | 中 | 检查物理增益是否生效；调大物理约束权重 |
| 光源老化漂移 | 低 | 自动校准每次启动跑一遍 |
| 新工艺适配 | 低 | 只需微调检测头，物理部分不用动 |

## 技术决策问答

**Q: 为什么选 RepViT 而不是 MobileNet 或 YOLO？**
A: RepViT 在 ARM 上比 MobileNetV3 快 13 倍，而且原生支持训练后融合（重参数化），推理速度与普通 3×3 卷积完全一致。YOLO 的 Anchor-based 设计对小缺陷不友好。

**Q: 为什么不做成两套独立模型（增强 + 检测）？**
A: 两个任务共享底层特征（边缘、纹理、角点），独立部署意味着两倍参数、两倍计算、两倍内存。实测表明联合训练比分别训练高 3-5 个 mAP 点。

**Q: 为什么用半监督而不是多找点标注？**
A: SiC/GaN 标注成本 100-200 元/张，且需要半导体专家逐像素标注。50 张标注 + CycleGAN 无监督预训练 = 10000+ 等效样本，性价比最优。

**Q: 这个方案和其他方案比怎么样？**
A: 其他方案要么模型太大（YOLO ~25M 参数）跑不动 ARM，要么需要大量标注数据（5000+ 张），要么串行部署增强+检测超过时间预算。本方案是唯一在 < 10M 参数、< 80ms 约束下同时完成增强+检测的方案。
